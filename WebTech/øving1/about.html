Questions

  1. The server is what holds the information that the client wants to access. For instance if I want to visit google.com I would have to send a request to the server to be allowed to access the content on that website. The server can also obtain information from the client if it wants to, often in the form of cookies so that it can store your info for your next visit.
  2. The client is the one who requests to obtain the info from the server. If it's a webpage it wants to visit, it will use a browser to access the server, but there are other types as well. If you have a network that doesn't use servers, it will request info from other clients, which means that the information is distributed across every client. This is called P2P or peer-to-peer.
  3. An IP address is a unique number for every computer on a network. It's the way computers know who the sender and receiver of data is, and where to send the packets. There are both local and public IP addresses, so that you can either connect to others on the web, or machines on your local network. It is also seperated into IPv4 and IPv6, where IPv4 is the most common, but sooner or later we will run out of IPv4 addresses and we will have to use IPv6 for new clients on the network.
  4. DNS stands for "Domain Name System" and it is how we can use URLs instead of IPs to connect to webpages. For instance if I want to connect to google.com I can just type that in the URL bar instead of having to remember an IPv4 address.
  5. The Internet is a network that consists of many computers communicating with each other. WWW or "The Web" is an application that runs on the Internet.
  6. HTML is a markup-language that tells the browser what the webpage should look like. It also tells it where to find the different files that should be shown, and where it can find the styling in the form of a CSS file.
